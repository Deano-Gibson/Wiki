# Delivery Standards

Delivery standards make projects easier to start, review, launch, and hand over.

## Baseline

Every meaningful project should have:

- Project brief
- Technical specification or implementation plan
- Setup instructions
- Environment variable documentation
- Deployment path
- Ownership and access notes
- Launch checklist
- Handover notes

## Delivery Principle

Every project should be understandable without asking the original builder to remember hidden context.

## Handover Minimum

- Repository access
- Production access
- Domain/DNS ownership
- Environment variable list without secret values
- Deployment instructions
- Known risks
- Maintenance tasks
- Support contacts
